## Description

Implement `TimeRange` and `Granularity` (PRD §3.2) in `Sources/StockChartsKit/Models/TimeRange.swift`. `TimeRange` mirrors the Robinhood chart selector exactly and drives every chart query.

## Scope

- `TimeRange: String, CaseIterable, Sendable, Codable` with cases `oneDay`=`1D`, `oneWeek`=`1W`, `oneMonth`=`1M`, `threeMonths`=`3M`, `yearToDate`=`YTD`, `oneYear`=`1Y`, `fiveYears`=`5Y`, `max`=`MAX`.
- `func interval(now: Date = .now, calendar: Calendar = .current) -> DateInterval`. For `max`, returns `(.distantPast, now)` (provider clamps to available history). YTD anchors at Jan 1 of `now`'s year.
- `var granularity: Granularity` — recommended sample hint (e.g. 1-minute for `1D`, daily for ≥`1M`).
- `Granularity: Sendable, Codable` with `oneMinute`, `fiveMinutes`, `fifteenMinutes`, `oneHour`, `oneDay`, `oneWeek`.

## Acceptance criteria

- Property-based / parameterized tests covering interval rollover edge cases: **DST transitions, year boundary, leap day** (PRD §12). Verify YTD computes Jan 1 correctly across years.
- `swift build` + `swift test` pass.

## References

PRD §3.2 (TimeRange/Granularity), §12 (DST/YTD/leap-day tests).

## Root cause

New implementation, no prior bug.

## Fix

Added `Sources/StockChartsKit/Models/TimeRange.swift`. `TimeRange` (String,
CaseIterable, Sendable, Codable) carries the exact raw values from §3.2.
`interval(now:calendar:)` subtracts a calendar component (day/month/year)
from `now` so DST, leap days, and year boundaries are handled by `Calendar`;
`yearToDate` anchors at Jan 1 of `now`'s year; `max` returns
`(.distantPast, now)`. A `min(start, now)` guard prevents an inverted
interval. `granularity` returns the per-range sample hint. `Granularity`
(Sendable, Codable) has the six cases; it also adopts Hashable (a superset of
the PRD signature) because `PerformanceSeries` is Hashable and contains a
`granularity`.

## Verification

Ran `swift build` (Build complete) and `swift test` — 26 tests in 4 suites
passed. TimeRange coverage in `TimeRangeTests.swift`: exact raw values and
8-case CaseIterable count, `max` start == `.distantPast`, YTD == Jan 1
parameterized across 2023–2026, one-day interval across the 2024
spring-forward DST transition (duration < 24h), one-year across a year
boundary, one-year and one-month into leap-day February 2024, granularity
hints, and Codable round-trip for all ranges plus a Granularity.
