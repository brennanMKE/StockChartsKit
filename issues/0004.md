## Description

Implement the remaining value types in `Sources/StockChartsKit/Models/` (PRD §3.3–§3.7). All `public`, `Sendable`, `Codable`, value types; money via `Money`, never `Double`.

## Scope

- `Account.swift`: `Account` (`Identifiable`), plus `AccountID`, `ProviderID`, `ConnectionID` (RawRepresentable wrappers) and `AccountKind` enum (§3.3).
- `Position.swift`: `Position`, `Symbol` (RawRepresentable), `AssetClass` enum (§3.4). Quantity is `Decimal` (fractional allowed); `averageCost`/`unrealizedPL` optional.
- `Balance.swift`: `Balance` with `total`, `cash`, optional `buyingPower`, `asOf` (§3.5).
- `Quote.swift`: `Quote` with `last`, optional `previousClose`, `asOf` (§3.6).
- `PerformanceSeries.swift`: `PerformanceSeries` with `subject`, `range`, `points`, `granularity`; nested `Point`, `Subject` enum (symbol/account/portfolio), and computed `summary: Summary?` plus `Summary` struct (§3.7).
- `summary` rules: `nil` when `points` is empty; `percentChange` is `nil` when `start.amount == 0`.

## Acceptance criteria

- Unit tests for `PerformanceSeries.summary`: empty points → nil summary; zero start → nil percentChange; normal case matches "$+290.85 (+145.35%)" style math.
- All types round-trip through `Codable`.
- `swift build` + `swift test` pass.

## References

PRD §3.3–§3.7.

## Root cause

New implementation, no prior bug.

## Fix

Added the remaining model files under `Sources/StockChartsKit/Models/`:
`Account.swift` (`Account` Identifiable, plus `AccountID`/`ProviderID`/
`ConnectionID` RawRepresentable wrappers and the `AccountKind` enum),
`Position.swift` (`Position` with `Decimal` quantity, `Symbol`, `AssetClass`),
`Balance.swift`, `Quote.swift`, and `PerformanceSeries.swift` (nested `Point`,
`Subject` enum, `Summary` struct, and the computed `summary`). `summary`
returns `nil` when `points` is empty, and `percentChange` is `nil` when
`start.amount == 0`; otherwise `percentChange = (end - start) / start`.
Absolute change is computed on the start currency directly to keep the
computed property non-throwing. All types are public, Sendable, Codable
value types and carry explicit public memberwise initializers.

## Verification

Ran `swift build` (Build complete) and `swift test` — 26 tests in 4 suites
passed. Coverage in `ModelTests.swift`: empty points → nil summary, zero
start → nil percentChange, single point → zero change, and the normal case
(200.10 → 490.95 = +290.85, percentChange rounds to 1.4535 / +145.35%);
plus Codable round-trips for Account (with and without connectionID),
Position, Balance, Quote, and PerformanceSeries across all three Subject
kinds.
