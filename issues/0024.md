## Description

Build the shared HTTP-replay test infrastructure (PRD §12) so every provider's tests run offline against recorded, redacted JSON fixtures. `swift test` must pass on macOS 14 with **no network access**.

## Scope

- A custom `URLProtocol` subclass that intercepts requests and replays canned responses from JSON fixtures.
- Fixture layout convention under `Tests/.../Fixtures/` keyed by request (path/method).
- Helpers to install/remove the protocol on a `URLSession`/`HTTPClient` for a test.
- **Redaction**: fixtures are recorded from real responses with all tokens/secrets/PII stripped. Do not commit live tokens (§12, §18). A redaction checklist/helper to keep contributors honest.

## Acceptance criteria

- A reference test using the replay protocol passes offline.
- Verified: running `swift test` with networking disabled still passes.
- This unblocks the replay tests in #0012, #0014, #0016, #0018, #0019, #0020, #0021.

## References

PRD §12 (HTTP replay, no live tokens, offline `swift test`).

## Root cause

New implementation, no prior bug.

## Fix

Built the shared offline HTTP-replay stack in `StockChartsKitTesting`:

- `ReplayURLProtocol` — a `URLProtocol` subclass that intercepts requests and
  serves a recorded `HTTPFixture` synchronously. It only claims requests
  carrying its session token header, so unrelated sessions are untouched. A
  request with no matching fixture fails with
  `ReplayError.noFixtureForRequest` rather than escaping to the network, so an
  un-recorded call is a loud failure, not a silent live request.
- `HTTPFixture` — `Codable` model (status / headers / body). Hand-authored
  fixtures may inline the body via a `bodyJSON` key.
- `FixtureStore` + `FixtureKey` — a loader keyed by HTTP method + path (+ an
  optional discriminator). Documented layout:
  `Tests/<ProviderTests>/Fixtures/<name>.json`, loaded via `Bundle.module` or an
  explicit file URL.
- Session helpers: `makeConfiguration(store:)` / `makeSession(store:)` install
  the protocol and bind it to a store via a per-session token header;
  `deregister(session:)` frees the binding.
- `Redactor` — scrubs known credential/PII JSON keys and literal values at any
  depth, plus `Redactor.checklist` for reviewers. No secrets in committed
  fixtures.
- Reference fixture `Tests/StockChartsKitTests/Fixtures/accounts.json`
  (declared as a `.copy` resource) and a reference test suite proving offline
  replay.

## Gotchas

`URLProtocol` is instantiated by the URL loading system, so it cannot capture a
`FixtureStore` reference directly. The store is held in a lock-guarded static
registry keyed by a UUID token that the install helper stamps onto the session
config's `httpAdditionalHeaders`; the protocol reads that header to find its
store. Tests must `deregister` (a `defer` in each test) to release the entry.
