## Description

Write the package root `README.md` (PRD §15, §16) documenting usage, capabilities, auth setup, caveats, and the secrets policy.

## Scope

1. One-paragraph description.
2. Quick-start example (PRD §16): register a Coinbase and a SnapTrade provider, aggregate, print total + 1-year history. Secrets come from the Keychain, never hardcoded.
3. Capabilities matrix (provider × capability).
4. Auth setup notes per provider, including where to register a developer app for E*Trade, Coinbase, Schwab, SnapTrade.
5. Caveats: snapshot store must be wound regularly; Yahoo fallback is unofficial (legal grey area); NetBenefits separate-login limitation; Tiingo free-tier quota.
6. Secrets policy (§5.1): developer-app credentials and per-user tokens supplied at runtime from the macOS Keychain, never hardcoded; app is local/non-distributed; shipping would require moving secrets behind a backend proxy first.

## Acceptance criteria

- README covers all six items; the quick-start example matches the actual public API and compiles conceptually against shipped types.
- No secrets anywhere in the README examples.

## Gotchas

Where the README corrects/simplifies vs PRD §16 and the real API:

- **`tokenStore` is separate from `snapshotStore`.** PRD §16 passed only `snapshotStore:`/`marketData:` to providers. The shipped providers take a `tokenStore:` (the `KeychainStore`, which conforms to each provider's `TokenStore` protocol) for per-user tokens, distinct from the optional `snapshotStore:`. The README passes `keychain` as `tokenStore`.
- **Coinbase & Schwab require a `"redirectURI"` secret** (Coinbase: `clientID` + `redirectURI`; Schwab: `clientID` + `clientSecret` + `redirectURI`). PRD §16 showed Coinbase with `clientID` only.
- **Provider initializers are `throws`** (they validate required config keys up front), so the README marks them `try`.
- **SnapTrade `discover` takes `tokenStore:` and assumes prior registration/connection.** The README documents the `registerUser` → `loginChallenge` → `discover` onboarding order rather than implying `discover` does everything.
- **Aggregator method is `combinedHistory(range:in:)`** — there is no `oneYearHistory`/`history(range:)` shortcut; the 1-year and 3-month examples both call `combinedHistory`.
- **`KeychainStore.secret(_:)` is package API**, not a host helper as PRD §16's comment implied (`secret(_:)`); the README clarifies this and drops the imagined host wrapper. (E\*Trade is omitted from the quick-start imports for brevity but fully documented in the auth section.)
