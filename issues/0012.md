## Description

Implement the `FXService` protocol and its default concrete `ECBFXService` (PRD §8). Used by `PortfolioAggregator` to convert when summing across currencies.

## Scope

- `protocol FXService: Sendable` with `rate(from:to:at:) async throws -> Decimal`.
- `ECBFXService`: hits the European Central Bank reference-rates endpoint (free, no auth) for fiat. For crypto, falls back to Coinbase's spot-price endpoint.
- Host may inject any other `FXService`.
- Network access through the shared `HTTPClient` (#0014) where practical; cache rates reasonably.

## Acceptance criteria

- Tests via HTTP replay fixtures (#0024) — no live network in `swift test`.
- Same-currency rate returns 1; cross-currency conversion produces expected `Decimal`.
- `swift build` + `swift test` pass.

## References

PRD §8 (aggregation / FXService).

## Root cause

New implementation, no prior bug.

## Fix

Added `Sources/StockChartsKit/Aggregation/FXService.swift`:

- `protocol FXService: Sendable` with `rate(from:to:at:) async throws -> Decimal`,
  plus a default `convert(_:to:at:)` Money helper.
- `ECBFXService` (the default concrete `FXService`):
  - Fiat via the ECB daily reference-rates **XML** feed
    (`eurofxref-daily.xml`), parsed with `Foundation.XMLParser` (no new
    dependency). The feed is EUR-based (`EUR -> X`), so cross rates are
    computed through EUR: `from -> to = (EUR -> to) / (EUR -> from)`, with
    `EUR -> EUR == 1` and inverses handled by division.
  - Crypto / unlisted pairs fall back to Coinbase's public spot-price JSON
    endpoint (`/v2/prices/{base}-{quote}/spot`).
  - Same currency returns `1`.
  - An unresolvable Coinbase pair (4xx `providerError`) maps to
    `BrokerageError.currencyMismatch`.
  - Networking goes through the shared `HTTPClient` over an injected
    `URLSession`, so tests replay fixtures with no live network.
  - Fetched ECB tables and spot prices are cached in an internal `actor`
    with a TTL (default 1h).

## Gotchas

- ECB rates are published as **XML, not JSON** — parsed with `XMLParser`;
  each `<Cube currency=.. rate=..>` element is one `EUR -> X` factor.
- Rates are EUR-based; non-EUR pairs MUST go through the EUR pivot
  (`(EUR->to)/(EUR->from)`), never a direct lookup.
- `at date:` is best-effort: the daily feed only carries the latest rates,
  so `date` is currently ignored and the latest rate is returned. Honoring
  history would require the ECB 90-day/full-history file (future
  enhancement).
