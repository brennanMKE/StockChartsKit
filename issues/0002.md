## Description

Implement `Money` and `CurrencyCode` (PRD §3.1) in `Sources/StockChartsKit/Models/Money.swift`. `Money` backs every monetary value in the package and must use `Foundation.Decimal`, never `Double`.

## Scope

- `Money`: `Hashable`, `Sendable`, `Codable`, value type. Fields `amount: Decimal`, `currency: CurrencyCode`.
- Initializers and helpers: `init(_:_:)` defaulting to `.usd`, `static func usd(_:)`, `static let zero`.
- Operators: `+` and `-` are **throwing** and throw `BrokerageError.currencyMismatch` on differing currencies; `*` by `Decimal` is non-throwing.
- `CurrencyCode`: `RawRepresentable` storing an uppercased ISO-4217-style string; `static let usd`. Supports crypto codes like `BTC`, `ETH`.
- **No internal rounding** — `Money` keeps full `Decimal` precision; rounding to minor units is a presentation concern. Equality/hashing on the exact stored `Decimal`.

## Acceptance criteria

- Property-based tests (`swift-testing`) on arithmetic: associativity where applicable, currency-mismatch throws, `*` precision preserved, `zero` identity.
- Depends on `BrokerageError.currencyMismatch` (#0005) for the throw — coordinate ordering or stub the error.
- `swift build` + `swift test` pass.
