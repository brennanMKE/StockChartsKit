## Description

Implement `Money` and `CurrencyCode` (PRD §3.1) in `Sources/StockChartsKit/Models/Money.swift`. `Money` backs every monetary value in the package and must use `Foundation.Decimal`, never `Double`.

## Scope

- `Money`: `Hashable`, `Sendable`, `Codable`, value type. Fields `amount: Decimal`, `currency: CurrencyCode`.
- Initializers and helpers: `init(_:_:)` defaulting to `.usd`, `static func usd(_:)`, `static let zero`.
- Operators: `+` and `-` are **throwing** and throw `BrokerageError.currencyMismatch` on differing currencies; `*` by `Decimal` is non-throwing.
- `CurrencyCode`: `RawRepresentable` storing an uppercased ISO-4217-style string; `static let usd`. Supports crypto codes like `BTC`, `ETH`.
- **No internal rounding** — `Money` keeps full `Decimal` precision; rounding to minor units is a presentation concern. Equality/hashing on the exact stored `Decimal`.

## Acceptance criteria

- Property-based tests (`swift-testing`) on arithmetic: associativity where applicable, currency-mismatch throws, `*` precision preserved, `zero` identity.
- Depends on `BrokerageError.currencyMismatch` (#0005) for the throw — coordinate ordering or stub the error.
- `swift build` + `swift test` pass.

## References

PRD §3.1 (Money), §14 (Decimal over Double), §12 (property-based tests).

## Root cause

New implementation, no prior bug.

## Fix

Added `Sources/StockChartsKit/Models/Money.swift` with `CurrencyCode`
(RawRepresentable, uppercases its raw value on init, `static let usd`) and
`Money` (Hashable, Sendable, Codable; `amount: Decimal`, `currency:
CurrencyCode`; `init(_:_:)` defaulting to `.usd`; `static func usd(_:)`;
`static let zero`). Operators `+`/`-` are throwing and throw
`BrokerageError.currencyMismatch` on differing currencies; `*` by `Decimal`
is non-throwing. No internal rounding — full `Decimal` precision is kept and
equality/hashing use the exact stored `Decimal`. Resolved together with
#0003/#0004/#0005 as one compile unit (Money's throw depends on
BrokerageError, which references the model wrapper types).

## Verification

Ran `swift build` (Build complete) and `swift test` — 26 tests in 4 suites
passed. Money coverage in `MoneyTests.swift`: currency-mismatch throws on
`+` and `-`, `*` preserves Decimal precision (0.1 * 0.3 == 0.03), `zero`
identity, equality on exact Decimal (1.0 == 1.00, distinct hash collapse),
parameterized same-currency arithmetic, and Codable round-trip.
