## Description

Implement a small reusable `HTTPClient` actor (PRD §11) that every provider wraps. No third-party HTTP client — `URLSession` with `data(for:)` only.

## Scope

- `actor HTTPClient` over `URLSession`.
- Single interception point for Bearer / OAuth signing (providers inject their signer).
- Retry-with-backoff on 429 and 5xx: max 3 attempts, exponential backoff; parse `Retry-After` header and surface `BrokerageError.rateLimited(retryAfter:)` when exhausted.
- Configured `JSONDecoder`: snake_case → camelCase, ISO 8601 dates, `Decimal` from string.
- Cancellation-aware: `Task.checkCancellation()` at entry; URLSession tasks cancelled via structured concurrency.
- Map transport/decoding failures to `BrokerageError.network` / `.decodingFailed`.

## Acceptance criteria

- Tests via a `URLProtocol` stub (#0024): retry fires on 429 then succeeds; `Retry-After` parsed; decoding of snake_case + Decimal-from-string works; cancellation propagates.
- `swift build` + `swift test` pass with no live network.

## References

PRD §11 (networking), §10 (error mapping), §13 (no third-party HTTP client).

## Root cause

New implementation, no prior bug.

## Fix

Added `Sources/StockChartsKit/Networking/HTTPClient.swift` — an `actor HTTPClient`
over `URLSession.data(for:)`:

- **Signing hook**: `RequestSigner` value wrapping
  `@Sendable (URLRequest) async throws -> URLRequest`, default `.identity`.
  Applied at one interception point on every attempt (re-signed each retry so
  timestamps/nonces stay fresh).
- **Retry/backoff**: `RetryPolicy` (max 3 attempts, exponential `base * 2^(n-1)`).
  Retries HTTP 429 and 5xx. Waits the larger of the backoff and the parsed
  `Retry-After`. Exhausted 429 → `BrokerageError.rateLimited(retryAfter:)`;
  other non-retryable HTTP → `providerError(providerID:statusCode:message:)`.
- **`Retry-After` parsing**: both delta-seconds and RFC 1123 HTTP-date forms.
- **Decoder**: `keyDecodingStrategy = .convertFromSnakeCase`, `.iso8601` dates.
- **Decimal-from-string**: `Sources/StockChartsKit/Networking/DecimalCoding.swift`
  adds `@LossyDecimal` property wrapper decoding a `Decimal` from a JSON number
  or a numeric string via `Decimal(string:locale: POSIX)` (exact base-10, no
  `Double`). Includes a `KeyedDecodingContainer` overload for optionals.
- **Cancellation**: `Task.checkCancellation()` at entry and per attempt;
  `URLError.cancelled`/`CancellationError` re-thrown as `CancellationError`.
- **Injectable** `sleep` and `now` closures plus `RetryPolicy.immediate` so the
  suite never waits on a real clock.

Extended `StockChartsKitTesting/FixtureStore.swift` minimally and
backward-compatibly: `register(key:sequence:)` (per-key FIFO of responses, last
one repeats once exhausted; takes precedence over a single fixture) and
`hitCount(for:)` (served-request count per key). Existing single-fixture
registration and all prior fixtures/tests are unchanged.

## Gotchas

- **Decimal-from-string**: `JSONDecoder` has no built-in strategy to accept a
  `Decimal` as both number and string, so model fields opt in with
  `@LossyDecimal`. String parse uses `Decimal(string:locale:)` (POSIX) for exact
  base-10 precision — never `Double`. Numbers decode directly as `Decimal`.
- **Replay-infra extension**: the store now supports a sequential response queue
  (`register(key:sequence:)`) because a single key previously returned one fixed
  response; the 429→200 retry test needs different responses on successive calls.
  Backward compatible — single-fixture path untouched.
- **Signing-hook shape**: `RequestSigner` is a `Sendable` struct around an async
  throwing closure (default `.identity`), applied to every attempt. Providers
  inject Bearer/OAuth signing here in one place.
- `providerID` is set per-client (constructor) and labels `providerError`.
