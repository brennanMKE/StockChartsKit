## Description

Implement `CSVImportProvider` in `Sources/StockChartsKitCSV/` (PRD §9.1) — the easiest provider, used to prove the protocol works and as a fallback for unsupported brokerages.

## Scope

- Reads positions / balances / snapshots from CSV files in a directory.
- Authentication: none (`authenticationStatus` → connected; `authenticate` is a no-op).
- Capabilities: `[.positions, .balances]`. `quote` and `priceHistory` throw `.unsupported`.
- Configurable column mapping per file (e.g. `AMD,12,490.70`).
- Watches the directory with FSEvents and reloads on change.

## Acceptance criteria

- Tests over sample CSV fixtures: parses positions/balances, applies custom column mapping, surfaces `.unsupported` for quote/priceHistory, reloads on file change.
- Conforms cleanly to `BrokerageProvider` — first end-to-end proof of the protocol.
- `swift build` + `swift test` pass.

## Fix

Replaced the `StockChartsKitCSV` stub with a real `CSVImportProvider` (an
`actor` conforming to `BrokerageProvider`) plus a small parsing layer:

- `CSVParsing` — dependency-free tokenizer: quoted fields, `""`-escaped quotes,
  embedded commas, surrounding whitespace, CRLF/CR normalisation, blank-line
  skipping.
- `ColumnMapping` — per-field column resolution that mixes positional
  (`AMD,12,490.70`) and header-based (`account,symbol,quantity,...`) columns,
  with `hasHeaderRow` and sensible defaults.
- `CSVPortfolioParser` — pure value type (no I/O, independently testable) that
  turns CSV text + a mapping into `[Position]` / `[AccountID: Balance]`.
  Decimals via `Decimal(string:)`, money stamped with the configured currency.
- `DirectoryWatcher` — `DispatchSource` vnode monitor on the directory fd;
  cancellable, `@unchecked Sendable`, closes the fd on cancel/deinit.
- `CSVImportProvider` — `nonisolated` `id` (`"csv"`), `displayName`
  (`"CSV Import"`), `capabilities = [.positions, .balances]`, injected
  `snapshotStore`/`marketData`. No-auth (`authenticationStatus` → `.connected`,
  `authenticate` → trivial complete `AuthSession`, `signOut` no-op).
  `quote`/`priceHistory` throw `.unsupported`. `listAccounts`/`positions`/
  `balance` serve parsed state. `reload()` reparses both files; the watcher
  callback simply calls `reload()`.

## Gotchas

- **CSV schema convention:** directory holds `positions.csv` and `balances.csv`
  (both names + mappings configurable). Positions default to the headerless
  `symbol,quantity,price` form with `marketValue = quantity × price`; a
  header-based mapping supports `account,symbol,quantity,marketValue,
  averageCost,assetClass`. Balances default to header-based
  `account,total,cash,buyingPower`. Rows without an account column fall back to
  the configurable `fallbackAccountID` (default `"csv"`).
- **FSEvents vs `reload()`:** reload logic is exposed as a public `reload()` and
  tested directly; the watcher only forwards events to it. Avoids flaky
  timing-dependent assertions.
- **Malformed rows:** never crash. A row missing required columns or with an
  undecodable `Decimal` is skipped with an `os.Logger` warning (subsystem
  `co.sstools.stockchartskit`, category `csv`) that logs only the line number —
  never field contents. Missing/unreadable files are treated as empty.
