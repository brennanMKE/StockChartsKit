## Description

Define the `SnapshotStore` actor protocol and `PortfolioSnapshot` value type in `Sources/StockChartsKit/Protocols/SnapshotStore.swift` (PRD §7). Providers that return only current state snapshot locally so the package can reconstruct historical portfolio-value series.

## Scope

- `protocol SnapshotStore: Actor` with `recordSnapshot(_:)`, `snapshots(accountID:in:)`, `snapshots(providerID:in:)`, `prune(olderThan:)`.
- `struct PortfolioSnapshot: Hashable, Sendable, Codable` with `accountID`, `timestamp`, `totalValue: Money`, `positions: [Position]` (§7).

## Acceptance criteria

- Compiles; `SQLiteSnapshotStore` (#0011) and `InMemorySnapshotStore` (#0015) conform.
- `swift build` passes.

## References

PRD §7 (snapshot store).

## Root cause

New implementation, no prior bug.

## Fix

Added `Sources/StockChartsKit/Protocols/SnapshotStore.swift`:

- `protocol SnapshotStore: Actor` with `recordSnapshot(_:)`,
  `snapshots(accountID:in:)`, `snapshots(providerID:in:)` (providerID optional,
  `nil` = all), and `prune(olderThan:)`.
- `struct PortfolioSnapshot: Hashable, Sendable, Codable` with `accountID`,
  `timestamp`, `totalValue: Money`, `positions: [Position]`, plus a memberwise
  `init`. Matches PRD §7.

The `SQLiteSnapshotStore` implementation and the `portfolioHistory` extension
are deliberately left to later issues (0010, 0011).

- `swift test` — Test run with 35 tests in 9 suites passed. The protocol and
  struct compile in the core target; conformances (`SQLiteSnapshotStore`,
  `InMemorySnapshotStore`) arrive in later issues (0011, 0015).
