## Description

Provide the default `portfolioHistory(accountID:range:)` implementation as a protocol extension on `BrokerageProvider` (PRD §7) so providers without native portfolio history get reconstruction for free from their injected `snapshotStore` + `marketData`.

## Scope

- `extension BrokerageProvider { public func portfolioHistory(accountID:range:) async throws -> PerformanceSeries }`.
- Throw `BrokerageError.unsupported(method: "portfolioHistory")` when both `snapshotStore` and `marketData` are nil.
- Algorithm: (1) pull snapshots in range from the store; (2) if sparse, interpolate by replaying positions × market prices — **symbols that don't resolve in market data are carried at their last known snapshot value, never dropped**; (3) resample to `range.granularity`; (4) return `PerformanceSeries`.
- Providers with `.nativePortfolioHistory` (none in v1) override this directly.

- Depends on #0007, #0008, #0009, #0015.

## Fix

Added `extension BrokerageProvider` with the default
`portfolioHistory(accountID:range:)` in
`Sources/StockChartsKit/Snapshot/PortfolioHistory.swift`. Throws
`BrokerageError.unsupported(method: "portfolioHistory")` when both
`snapshotStore` and `marketData` are nil. Algorithm: (1) pull snapshots for the
account in `range.interval()` from the store; (2) prefetch each held symbol's
`marketData.priceHistory` once (symbols whose fetch throws are omitted from the
price map, signalling carry-forward); (3) build a value point at each resample
timestamp stepped by `range.granularity`, taking the nearest-prior snapshot and
revaluing each position as `quantity × nearest-prior market price`; positions
whose symbol has no usable market price are carried at their stored snapshot
`marketValue` (never dropped); a position-less snapshot uses its `totalValue`;
(4) return a `PerformanceSeries` with `subject = .account(accountID)` and
`granularity = range.granularity`.

## Gotchas

- **Resampling choice**: fixed step from `range.granularity` with nearest-prior
  (last-observation-carried-forward / step-function) lookup for both snapshots
  and market prices. Resample timestamps before the first snapshot are skipped
  so no pre-history values are fabricated; the series starts at the first real
  data point. A `.distantPast`-anchored interval (`.max`) is capped at 5000
  samples so reconstruction stays finite.
- **Carry-forward**: a symbol whose `marketData.priceHistory` throws (or yields
  no point at/before the timestamp) is revalued at the position's stored
  snapshot `marketValue`, never dropped — verified explicitly by the
  unresolved-symbol test.
- **No-store path**: when only `marketData` is non-nil (no store), there are no
  account positions to reconstruct, so an empty series is returned rather than
  throwing — `.unsupported` is reserved for the both-nil case per PRD §7.
- **providerID mapping**: reconstruction queries by `accountID`, so the store's
  provider map is not needed here.
