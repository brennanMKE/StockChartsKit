## Description

Implement `BrokerageError` (PRD §10) in `Sources/StockChartsKit/Errors/BrokerageError.swift`. Every provider maps its native errors into one of these cases. This is a foundational type that `Money` (#0002) and the protocols depend on, so it should land early.

## Scope

- `enum BrokerageError: Error, Sendable` with cases: `notAuthenticated`, `authenticationFailed(reason:)`, `authorizationRevoked`, `rateLimited(retryAfter: Date?)`, `unsupported(method:)`, `currencyMismatch(lhs: CurrencyCode, rhs: CurrencyCode)`, `providerError(providerID: ProviderID, statusCode: Int?, message:)`, `decodingFailed(underlying: Error)`, `network(underlying: Error)`, `missingSymbol(Symbol)`, `stale(asOf: Date)`.
- Depends on `CurrencyCode`, `ProviderID`, `Symbol` types (#0002, #0004) — coordinate ordering.

## Acceptance criteria

- Compiles; cases match PRD §10 exactly.
- `swift build` passes.
