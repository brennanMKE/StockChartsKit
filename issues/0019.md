## Description

Implement `ETradeProvider` in `Sources/StockChartsKitETrade/` (PRD §9.3) against `https://api.etrade.com`. The hard part is OAuth 1.0a (HMAC-SHA1 signed requests).

## Scope

- `OAuth1Signer.swift`: HMAC-SHA1 request signing using **swift-crypto** — no third-party OAuth lib (§13). Prefer typed throws for the signing helper (§14).
- Three-legged OAuth 1.0a flow:
  1. `GET /oauth/request_token` (signed with consumer secret).
  2. Present `https://us.etrade.com/e/t/etws/authorize?key={ck}&token={rt}` via `AuthChallenge.pinCode`.
  3. User pastes verifier PIN → `AuthSession.complete(pinCode:)`.
  4. `GET /oauth/access_token` exchanges request token + PIN.
  5. Access tokens expire at midnight US Eastern; refresh via `GET /oauth/renew_access_token`.
- Reads: `listAccounts` → `/v1/accounts/list.json`; `positions` → `/v1/accounts/{accountIdKey}/portfolio.json`; `balance` → `/v1/accounts/{accountIdKey}/balance.json?instType=BROKERAGE&realTimeNAV=true`; `quote` → `/v1/market/quote/{symbol}.json`.
- `priceHistory` throws `.unsupported` (E*Trade has none; host falls through to MarketDataProvider).
- Capabilities: `[.positions, .balances, .quotes]`.
- Sandbox base URL `https://apisb.etrade.com`; expose production vs sandbox via `Configuration`/`environment` initializer.

## Acceptance criteria

- Unit tests for `OAuth1Signer` against known HMAC-SHA1 test vectors (signature base string + key construction).
- HTTP replay tests for accounts/portfolio/balance/quote; `priceHistory` returns `.unsupported`.
- `consumerKey`/`consumerSecret` via `Configuration` from Keychain — never hardcoded, never logged.
- `swift build` + `swift test` pass with no live network.

## Fix

Implemented the `StockChartsKitETrade` target (replacing the stub):

- `OAuth1Signer.swift` — OAuth 1.0a / HMAC-SHA1 signer using `import Crypto`
  (swift-crypto's `HMAC<Insecure.SHA1>`). Builds the RFC 5849 §3.4.1.1 signature
  base string (`METHOD&pct(baseURI)&pct(sorted merged params)`), signs with key
  `pct(consumerSecret)&pct(tokenSecret)`, base64-encodes the digest, and emits
  the `Authorization: OAuth …` header. RFC 3986 §2.3 percent-encoding
  (unreserved literal, uppercase hex). Nonce and timestamp are injectable for
  deterministic tests. Typed `throws(OAuth1Error)` — the error space is closed
  (only `.invalidURL`).
- `ETradeProvider` — `actor` conforming to `BrokerageProvider`. `environment`
  (`.production` / `.sandbox`) selects the base URL. Three-legged OAuth 1.0a PIN
  flow: `GET /oauth/request_token` (callback `oob`) → present
  `https://us.etrade.com/e/t/etws/authorize?key=…&token=…` via
  `AuthChallenge.pinCode` → `complete(pinCode:)` calls `GET /oauth/access_token`
  with `oauth_verifier` and persists the token/secret via the `TokenStore`
  pattern. `renewAccessToken()` implements `GET /oauth/renew_access_token`.
  Reads (`list`/`portfolio`/`balance`/`quote`) are OAuth1-signed by an
  `HTTPClient` `RequestSigner` that loads the stored token/secret at send time;
  `priceHistory` throws `.unsupported`. `consumerKey`/`consumerSecret` come from
  `Configuration`; never hardcoded or logged.
- `ETradeTokens.swift` — persisted token bundle + JSON wire types. E*Trade nests
  responses under PascalCase keys (`AccountListResponse`, `Computed`, `All`,
  `QuoteData`, …) that `.convertFromSnakeCase` does not touch, so each is mapped
  with explicit `CodingKeys`. All money/quantity decoded as `Decimal`.
- `TokenStore.swift` — the same `TokenStore` protocol the Coinbase provider uses
  (each provider target declares its own copy; `KeychainStore` conforms).

## Gotchas

- **Signature base string encoding**: the parameter string must be
  percent-encoded *twice* in effect — each key/value is encoded once, joined
  with `&`/`=`, then the whole parameter string is encoded again when folded
  into the base string. Only the RFC 3986 unreserved set stays literal, and
  hex must be uppercase (`addingPercentEncoding` already uppercases). The
  base-string URI omits the query/fragment and the default port. `oauth_version`
  is included by the signer; the RFC example happens to omit it, so the vector
  test calls the base-string builder with explicit parameters.
- **accountId vs accountIdKey**: the accounts list returns both; portfolio and
  balance paths use the opaque `accountIdKey`, so `Account.id` is mapped from
  `accountIdKey` (not the human-facing `accountId`).
- **Token expiry / renew**: E*Trade access tokens expire at midnight US Eastern
  and go inactive after two hours; there is no issued `expiresAt`, so the bundle
  stores only token+secret and renewal is on demand via
  `GET /oauth/renew_access_token`.
- **Form-encoded auth responses**: the request/access-token legs return
  `application/x-www-form-urlencoded` bodies (not JSON), parsed with a small
  form decoder; fixtures store them as base64 `body` (not `bodyJSON`, which
  would re-encode a string with JSON quotes).
- **PascalCase JSON keys**: `.convertFromSnakeCase` does not lowercase
  PascalCase, so the wire structs need explicit `CodingKeys`.
