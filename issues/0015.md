## Description

Implement the `StockChartsKitTesting` library target (PRD §12) providing test doubles used across the core and provider test suites. Landing this early unblocks tests for #0010, #0013, and others.

## Scope

- `MockBrokerageProvider`: a programmable `BrokerageProvider` conformance whose responses (accounts, positions, balances, quotes, histories, auth status) are configurable per test.
- `InMemorySnapshotStore`: an in-memory `SnapshotStore` for unit tests.
- `FixedMarketDataProvider`: returns canned `Quote` / `PerformanceSeries`.

## Acceptance criteria

- All three conform to their protocols and are `Sendable`-clean under strict concurrency.
- Smoke tests exercising each double.
- `swift build` + `swift test` pass.

## References

PRD §12 (testing support library).

## Root cause

New implementation, no prior bug.

## Fix

Replaced the `StockChartsKitTesting` stub with three test doubles, all
`Sendable`-clean under Swift 6 strict concurrency:

- `MockBrokerageProvider` — a programmable `actor` conforming to
  `BrokerageProvider`. `id` / `displayName` / `capabilities` / `snapshotStore`
  / `marketData` are `nonisolated` and set at init; every protocol method is
  driven by a settable `@Sendable` closure with `set…` helpers, so tests inject
  canned values or throwing behaviour. Defaults are empty/`.unsupported`.
  `portfolioHistory` surfaces a settable override (defaults to
  `.unsupported`); the reconstruction algorithm is deferred to #0010.
- `InMemorySnapshotStore` — an `actor` conforming to `SnapshotStore`, backed by
  a timestamp-sorted array, with `recordSnapshot`, both `snapshots(…)` queries,
  and `prune(olderThan:)`.
- `FixedMarketDataProvider` — a `Sendable` `struct` conforming to
  `MarketDataProvider`, returning a canned `Quote`/`PerformanceSeries`
  (configurable price/granularity, defaulting to `$100`).

## Gotchas

A `PortfolioSnapshot` carries an `accountID` but no provider identity, so
`InMemorySnapshotStore.snapshots(providerID:in:)` cannot derive a provider from
a snapshot alone. The store therefore keeps an injectable
`accountID -> providerID` map: seed it via `init(accountProviders:)`, add pairs
with `mapAccount(_:to:)` / `register(_ account:)`, or use the
`recordSnapshot(_:account:)` convenience to learn it. Snapshots whose account is
unmapped are excluded from a non-`nil` `providerID` query and included when
`providerID` is `nil`.
