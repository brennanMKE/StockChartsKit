## Description

Implement `SchwabProvider` in `Sources/StockChartsKitSchwab/` (PRD §9.4) against `https://api.schwabapi.com`. OAuth 2.0, much cleaner than E*Trade; Schwab supports native price history.

## Scope

- Authentication: OAuth 2.0 with client credentials registered via the Schwab developer portal; drive via `AuthChallenge.oauthRedirect`. Client ID/secret via `Configuration` from Keychain.
- Endpoints under `https://api.schwabapi.com/trader/v1/` for accounts/positions/balances/quotes.
- `priceHistory` supported natively via `/marketdata/v1/pricehistory`.
- Capabilities: `[.positions, .balances, .quotes, .nativePriceHistory]`.
- Use shared `HTTPClient` (#0014); map errors to `BrokerageError`.

## Acceptance criteria

- No secrets in source/fixtures; tokens never logged.
- `swift build` + `swift test` pass with no live network.

## Fix

Implemented `SchwabProvider` (an `actor` conforming to `BrokerageProvider`) in
`Sources/StockChartsKitSchwab/`, mirroring the Coinbase OAuth2 provider shape
(token-store protocol + actor token-refresh collaborator + Bearer request
signer), adapted to Schwab's specifics:

- **Auth — OAuth 2.0 authorization-code.** `authenticate()` vends a
  `SchwabAuthSession` driving `AuthChallenge.oauthRedirect`. The authorization
  URL carries `response_type=code`, `client_id`, `redirect_uri`, `scope=readonly`
  and an opaque `state`. `complete(callbackURL:)` validates `state`, extracts
  `code`, and exchanges it at the token endpoint. **Schwab authenticates the
  client with HTTP Basic auth** (`Authorization: Basic base64(clientID:clientSecret)`)
  for both the code exchange and refresh (`grant_type=refresh_token`) — not via
  body credentials. `clientID`/`clientSecret`/`redirectURI` come from
  `Configuration` secrets; tokens persist via the `TokenStore`/`KeychainStore`
  pattern. `authenticationStatus()` returns `.connected` when a valid or
  refreshable token exists, else `.requiresChallenge(.oauthRedirect(...))`.
- **Reads** (Bearer-signed via `HTTPClient` `RequestSigner`): `listAccounts()`
  joins `GET /trader/v1/accounts/accountNumbers` with `GET /trader/v1/accounts`;
  `positions(for:)` → `GET /trader/v1/accounts/{hash}?fields=positions`;
  `balance(for:)` from `currentBalances`; `quote(for:)` → `GET /marketdata/v1/quotes`.
- **Native price history**: `priceHistory(symbol:range:)` → `GET
  /marketdata/v1/pricehistory` with `periodType`/`period`/`frequencyType`/`frequency`
  mapped from `TimeRange`. `capabilities = [.positions, .balances, .quotes,
  .nativePriceHistory]`.

Wire money/quantity fields decode through the package's `LossyDecimal` wrapper so
string-encoded decimals keep full precision (never `Double`).

## Gotchas

- **TimeRange → pricehistory mapping.** Schwab intraday data requires
  `periodType=day`, so short ranges use minute candles and a month+ uses
  daily/weekly/monthly: 1D=day/1/minute/1, 1W=day/5/minute/30, 1M=month/1/daily/1,
  3M=month/3/daily/1, YTD=ytd/1/daily/1, 1Y=year/1/daily/1, 5Y=year/5/weekly/1,
  MAX=year/20/monthly/1. The returned `granularity` is `range.granularity`; the
  chart layer downsamples as needed.
- **Basic-auth token exchange.** Unlike Coinbase (PKCE, no secret), Schwab
  authenticates the client with `Authorization: Basic base64(clientID:clientSecret)`
  on both the code exchange and refresh — the client id/secret are NOT in the body.
- **Account number vs. hash.** `GET /accounts/accountNumbers` maps the visible
  `accountNumber` to an opaque `hashValue`. `listAccounts()` surfaces the **hash**
  as the `AccountID` so per-account paths work directly (mirrors E*Trade's
  `accountIdKey`); the display name keeps the visible number.
- **Token refresh.** The Bearer signer fetches a valid token per request via
  `SchwabTokenProvider` (an actor), refreshing in place when the access token is
  within 60s of expiry, keeping the prior refresh token if the issuer omits a new one.
