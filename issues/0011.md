## Description

Implement `SQLiteSnapshotStore` in `Sources/StockChartsKit/Snapshot/SQLiteSnapshotStore.swift` (PRD §7), the default `SnapshotStore`, backed by GRDB.swift.

## Scope

- `actor SQLiteSnapshotStore` conforming to `SnapshotStore`, `init(url:) throws`.
- Schema v1 per PRD §7: `snapshot` table (`account_id`, `timestamp` REAL epoch, `total_value` TEXT Decimal-as-string, `currency`, PK `(account_id, timestamp)`), index `snapshot_ts`, and `snapshot_position` table (`account_id`, `timestamp`, `symbol`, `quantity`, `market_value`, `currency`, PK `(account_id, timestamp, symbol)`).
- Use GRDB `DatabaseMigrator` (or `PRAGMA user_version`). v1 is the schema above; future changes ship as new ordered migrations — **never mutate an existing migration**. Run pending migrations on init so existing DBs upgrade in place.
- `Decimal` serialized/deserialized as string (no `Double`).
- Implement `recordSnapshot`, `snapshots(accountID:in:)`, `snapshots(providerID:in:)`, `prune(olderThan:)`.

## Acceptance criteria

- Tests: record + read-back round trip, interval filtering, prune, Decimal precision preserved through string serialization, migrator runs cleanly on a fresh DB.
- `swift build` + `swift test` pass with no network.

## References

New implementation, no prior bug.

## Fix

Added `actor SQLiteSnapshotStore` in
`Sources/StockChartsKit/Snapshot/SQLiteSnapshotStore.swift`, backed by GRDB
(`DatabaseQueue`). Schema v1 created via `DatabaseMigrator` exactly per PRD §7:
`snapshot(account_id TEXT, timestamp REAL, total_value TEXT, currency TEXT,
PK(account_id, timestamp))`, index `snapshot_ts ON snapshot(timestamp)`, and
`snapshot_position(account_id, timestamp, symbol, quantity, market_value,
currency, PK(account_id, timestamp, symbol))`. `init(url:)` opens/creates the DB
and runs pending migrations. `recordSnapshot` writes the snapshot row plus its
position rows in a single `write` transaction (INSERT OR REPLACE, clearing stale
position rows first for idempotent re-records). `Decimal` values serialize to
exact base-10 `TEXT` (locale `en_US_POSIX`), never `Double`; timestamps are Unix
epoch `REAL`. `snapshots(accountID:in:)`, `snapshots(providerID:in:)`, and
`prune(olderThan:)` implemented; provider filtering uses an injectable
`accountID -> providerID` map (constructor seed + `mapAccount`/`register`),
mirroring `InMemorySnapshotStore`.

## Gotchas

- **Migrations are append-only**: the type doc and `migrator` comment state that
  `v1` must never be mutated; future schema changes ship as new ordered
  migrations so existing user DBs upgrade in place.
- **providerID mapping**: the schema has no provider column (matching
  `PortfolioSnapshot`), so `snapshots(providerID:)` resolves provider via an
  injectable `accountID -> providerID` map. A non-nil providerID matches only
  mapped accounts (unmapped excluded); nil matches all — consistent with
  `InMemorySnapshotStore`.
- **assetClass not persisted**: the PRD §7 `snapshot_position` schema has no
  asset-class column, so positions hydrate with `assetClass = .other`. The
  reconstruction algorithm only needs symbol, quantity, and market value, so
  this is lossless for the store's purpose.
