## Description

Implement `PortfolioAggregator` in `Sources/StockChartsKit/Aggregation/PortfolioAggregator.swift` (PRD §8) — aggregates accounts, total value, and combined history across multiple providers, doing FX conversion when summing across currencies.

## Scope

- `actor PortfolioAggregator`, `init(providers:snapshotStore:marketData:fx:)` with `fx` defaulting to `ECBFXService()`.
- `allAccounts() async throws -> [Account]` — union across providers.
- `totalValue(in:) async throws -> Money` — sum balances, FX-converting via injected `FXService`.
- `combinedHistory(range:in:) async throws -> PerformanceSeries` — merge per-account/portfolio histories into one series in the target currency.
- `liveTotalValues(in:) -> AsyncThrowingStream<Money, Error>` — streams partial totals as providers finish loading, so the macOS app can show partial results while slow providers (Fidelity) catch up.

## Acceptance criteria

- Tests using `MockBrokerageProvider` (#0015), `InMemorySnapshotStore`, `FixedMarketDataProvider`, and a stub `FXService`: total across two currencies converts correctly; `liveTotalValues` emits incremental updates and finishes; combined history aligns timestamps.
- Depends on #0012, #0015. `swift build` + `swift test` pass.

## Fix

Added `Sources/StockChartsKit/Aggregation/PortfolioAggregator.swift`:

- `actor PortfolioAggregator`, `init(providers:snapshotStore:marketData:fx:)`
  with `fx` defaulting to `ECBFXService()`.
- `allAccounts()` — unions accounts across providers, in provider order.
- `totalValue(in:)` — sums each account's `Balance.total`, FX-converting to
  the target currency via the injected `FXService`. Full `Decimal`
  precision, no rounding.
- `combinedHistory(range:in:)` — obtains each account's `portfolioHistory`
  (the snapshot-reconstruction default extension), builds the union of all
  series timestamps as the merged axis, and at each timestamp samples every
  account with nearest-prior (last-observation-carried-forward) lookup,
  FX-converting and summing. `subject = .portfolio(providerID: nil)`.
- `liveTotalValues(in:)` — `nonisolated`, returns an
  `AsyncThrowingStream<Money, Error>`. Each provider's contribution is
  computed concurrently in a task group; as each completes the running
  cumulative total is yielded, and the stream finishes when all are done.

## Gotchas

- `liveTotalValues` error handling: a single provider failure is **surfaced
  but not fatal** — it is logged and contributes `0` while every other
  provider's contribution is preserved and the stream still finishes
  normally. This matches the PRD's "show partial results while slow
  providers catch up" intent rather than collapsing the whole view on one
  bad brokerage. (A stricter injected `FXService` whose conversion fails in
  the synchronous `totalValue(in:)` path will still propagate there.)
- `combinedHistory` alignment: the merged time axis is the **union** of all
  per-account timestamps (sorted, de-duplicated). Each account is sampled
  with nearest-prior carry-forward; accounts with no point yet at a
  timestamp contribute nothing (not started), so a newer account does not
  retroactively inflate earlier points.
- `liveTotalValues` is `nonisolated` so it can be consumed with a plain
  `for try await` from outside the actor; it captures the immutable `let`
  dependencies into the stream's task.
