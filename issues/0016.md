## Description

Implement the default `MarketDataProvider` stack in `Sources/StockChartsKitMarketData/` (PRD §6): a Tiingo primary, a Yahoo Finance fallback, and a tiered composite that caches daily bars to stay under quota.

## Scope

- `TiingoMarketData(apiKey:)` — primary, `https://api.tiingo.com`. API token supplied at runtime (from Keychain by host), never hardcoded. Best for daily charts.
- `YahooFinanceMarketData()` — fallback, `https://query1.finance.yahoo.com/v8/finance/chart/{symbol}`. Undocumented, no auth; treat as best-effort. Document the legal grey area in README (#0023).
- `TieredMarketDataProvider(primary:fallback:)` — tries Tiingo first, falls back to Yahoo on any failure.
- **Quota & caching**: cache fetched daily bars locally (reuse the snapshot SQLite DB or a sibling table) and serve cache-first for daily granularity; only hit the network for missing/stale ranges. Intraday (`1D`) fetched live.
- Map both ranges and granularity from `TimeRange`.

## Acceptance criteria

- HTTP replay tests (#0024) for both Tiingo and Yahoo response shapes; tiered provider falls back on primary failure; cache serves daily without a second network hit.
- No secrets in fixtures or source (§5.1, §18).
- `swift build` + `swift test` pass with no live network.

## Fix

Replaced the `StockChartsKitMarketData` stub with the default market-data stack
(PRD §6):

- `TiingoMarketData(apiKey:…)` — primary. Daily prices from
  `GET /tiingo/daily/{symbol}/prices` (mapped from `TimeRange` to a start/end
  date range, `MAX` omits `startDate`); quote from `GET /iex/{symbol}`. The API
  token is supplied at init by the host and sent via a `RequestSigner` as an
  `Authorization: Token …` header — never hardcoded, never logged, kept out of
  the URL (and therefore out of fixtures). Builds a daily-granularity
  `PerformanceSeries` of `Decimal` closes.
- `YahooFinanceMarketData()` — fallback. Parses the chart JSON at
  `GET /v8/finance/chart/{symbol}` (parallel `timestamp` + `indicators.quote[0].close`
  arrays) into a `Quote`/`PerformanceSeries`. `TimeRange` maps to Yahoo's
  `interval`/`range` query params. Unauthenticated, best-effort.
- `TieredMarketDataProvider(primary:fallback:)` — default composite. Tries the
  primary and falls back to the fallback on ANY error (quotes and history).
- Caching: a `DailyBarCache` protocol with an actor-backed in-memory default
  (`InMemoryDailyBarCache`). The tiered provider serves daily granularity
  cache-first; a range whose newest cached bar reaches the most recent expected
  trading day is served from cache with no network call, otherwise bars are
  fetched, cached, and returned. Intraday (`1D`) and weekly-sampled ranges
  bypass the cache and fetch live each call. `asOf` is carried by the series'
  newest point, so stale data is never presented as fresh.

Added the `StockChartsKitMarketDataTests` test target to `Package.swift`
(depends on `StockChartsKitMarketData` + `StockChartsKitTesting`, `Fixtures/`
copied as a resource) with seven offline replay tests and synthetic fixtures.

## Gotchas

- **Cache backing.** Shipped an in-memory `DailyBarCache` default rather than
  the GRDB sibling table. GRDB was NOT added to the `StockChartsKitMarketData`
  target — the in-memory default fully satisfies the cache-first behavior and
  tests, and the protocol lets a host inject a persistent (e.g. SQLite-backed)
  cache later without API churn. PRD §6 allows either ("reuse the snapshot
  SQLite DB or a sibling table"); persistence is documented as optional in the
  issue brief.
- **Yahoo legal grey area.** The endpoint is undocumented and unauthenticated;
  documented in the source DocC as best-effort. The README note is tracked
  separately under #0023.
- **Granularity/range mapping.** Tiingo daily covers `1M`+ ranges; Yahoo uses
  per-range `interval`/`range` pairs mirroring `TimeRange.granularity`. Only
  `oneDay`-granularity ranges use the daily cache; `1D` (intraday) and the
  weekly-sampled `5Y`/`MAX` bypass it and fetch live.
- **`asOf` handling.** The series carries `asOf` implicitly via its newest
  point's timestamp; cache coverage requires the newest bar to reach the most
  recent expected trading day (a weekday proxy that ignores holidays and errs
  toward a refetch), so cached data is never served when fresher bars are due.
- **Toolchain `Task.sleep(.zero)` quirk.** Retrying a replayed 5xx with a real
  (even zero) `Task.sleep` between attempts triggered a malloc nano-zone
  corruption ("freed pointer was not the last allocation") in this toolchain —
  reproducible with the bare shared `HTTPClient`, masked under ASan. Mirrored
  the existing `HTTPClientTests` convention: the providers now expose an
  injectable `sleep` (defaulting to `Task.sleep`) and the tests inject a no-op,
  so retries are still exercised without touching a clock. `DateFormatter`/
  `ISO8601DateFormatter` thread-safety was also addressed (lock-guarded parsing;
  date-string formatting via a UTC `Calendar`).
