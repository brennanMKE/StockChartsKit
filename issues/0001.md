## Description

`Package.swift` currently declares only the core `StockChartsKit` target and its test target. It must declare every target and product the PRD calls for (PRD §2, §13), one product per provider so the host app can pick which to link, with strict concurrency enabled and pinned dependencies.

## Scope

- `swift-tools-version:6.1` (or later already in repo), platforms `.macOS(.v14)`.
- Enable strict concurrency (`SwiftSettings.enableUpcomingFeature("StrictConcurrency")`).
- Targets: `StockChartsKit` (core, only depends on `swift-crypto` + `swift-collections`), `StockChartsKitTesting`, `StockChartsKitMarketData`, `StockChartsKitCSV`, `StockChartsKitCoinbase`, `StockChartsKitETrade`, `StockChartsKitSchwab`, `StockChartsKitSnapTrade`.
- One library product per provider + core + market data + testing.
- Matching test targets per PRD §2: `StockChartsKitTests`, `StockChartsKitETradeTests`, `StockChartsKitCoinbaseTests`, `StockChartsKitSnapTradeTests`, `StockChartsKitSchwabTests`, `StockChartsKitCSVTests`.
- Dependencies (pinned, PRD §13): `apple/swift-crypto`, `groue/GRDB.swift`, `apple/swift-collections`, `swift-testing` (toolchain-provided). No Alamofire, KeychainAccess, SwiftJWT, Vapor.
- Create the empty source directory skeleton from PRD §2 so the targets resolve.

## Acceptance criteria

- `swift build` resolves all targets (empty/stub sources are fine at this stage).
- Core target has no provider-specific or third-party deps beyond `swift-crypto` + `swift-collections`.
