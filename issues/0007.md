## Description

Define the core `BrokerageProvider` actor protocol in `Sources/StockChartsKit/Protocols/BrokerageProvider.swift` (PRD §4) — the single protocol every backend conforms to. Also defines `Capabilities` and `AuthenticationStatus`.

## Scope

- `protocol BrokerageProvider: Actor` with `nonisolated` pure properties `id: ProviderID`, `displayName: String`, `capabilities: Capabilities`, and `nonisolated` injected deps `snapshotStore: (any SnapshotStore)?`, `marketData: (any MarketDataProvider)?`.
- Auth methods: `authenticationStatus()`, `authenticate() -> AuthSession`, `signOut()`.
- Read methods: `listAccounts()`, `positions(for:)`, `balance(for:)`, `quote(for:)`, `priceHistory(symbol:range:)`, `portfolioHistory(accountID:range:)`.
- `Capabilities: OptionSet, Sendable, Codable` with `positions`, `balances`, `quotes`, `nativePriceHistory`, `nativePortfolioHistory`, `crypto`, `realtimeStreaming` (§4).
- `AuthenticationStatus: Sendable` with `connected(connectionID:)`, `requiresChallenge(AuthChallenge)`, `error(BrokerageError)`.
- Depends on protocols #0008 / #0009 for the injected dep types — coordinate ordering (forward refs are fine since they're separate files).

## Acceptance criteria

- Protocol surface matches PRD §4 exactly, keeps `id`/`displayName`/`capabilities` nonisolated so hosts can list providers without actor hops.
- `swift build` passes.
